# PLOT STRUCTURE: 剧情结构设计

> **W3专用。依赖W1(_Setting.md Part 1) + W2(_Setting.md Part 2)。**

---

## 1. 骨架设计

### 1.1 全局弧线

从核心前提推导：故事从 A 到 B，中间经过什么？

- **起点状态:** 角色/世界在故事开始时是什么样
- **终点状态:** 核心前提完全兑现时是什么样
- **不可逆节点:** 哪些事件发生后无法回头？

### 1.2 卷/章结构

- 每卷的核心问题是什么？
- 卷与卷之间的承接关系？
- 每卷的结尾悬念？

---

## 2. 弧线与锚点

### 角色弧线
每个主要角色的变化轨迹：起点 → 转折点 → 终点

### 锚点行为分配
从 CLAUDE.md 锚点行为定义中，为关键角色分配1-2个锚点行为。

---

## 3. 事件流

### 3.1 因果链
每个事件必须有前因和后果。随机事件 = 叙事缺陷。

### 3.2 Chunk展开

将每卷的事件流切分为 Chunks：

```markdown
### Chunk [N]: [标题]
- **锚点:** 时间: / 地点: / 在场:
- **状态流:** 起点: / 终点:
- **强度边界:** Tier: / 上限: / 禁止:
- **焦点:** 核心: / 基调: / 情绪目标:
- **必经节点:**
  - [节点1]
  - [节点2]
- **锚点行为:** 行为: / 本Chunk表现:
- **禁区:** [不可触碰的内容]
```

---

## 4. 质量门控

- [ ] 每个事件都能追溯到核心前提
- [ ] 因果链无断裂
- [ ] 每个Chunk有明确的Entry/Exit状态
- [ ] 锚点行为分配合理
- [ ] 用户确认
